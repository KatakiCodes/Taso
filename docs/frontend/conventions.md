# Convenções Frontend

Sempre utilizar Standalone Components.

Sempre utilizar inject().

Nunca utilizar constructor injection.

Nunca utilizar any.

Sempre tipar tudo.

Utilizar interfaces.

Preferir readonly.

Utilizar Signals para estado local.

Utilizar RxJS apenas para comunicação HTTP.

Sempre utilizar @if.

Sempre utilizar @for.

Sempre utilizar @switch.

Nunca utilizar \*ngIf.

Nunca utilizar \*ngFor.

Nunca utilizar módulos.

Componentes menores que 250 linhas.

Serviços menores que 300 linhas.

Evitar lógica de negócio nos componentes.

Utilizar Smart Components apenas quando necessário.

Preferir Dumb Components.

Utilizar aliases de import.

Organizar imports.

Componentes reutilizáveis deverão permanecer na pasta shared/ui.

Componentes específicos deverão permanecer dentro da feature.

Todo estilo deverá utilizar TailwindCSS.

Não utilizar CSS inline.

Não utilizar Bootstrap.

Não utilizar Angular Material.

Não utilizar PrimeNG.

Não utilizar bibliotecas de UI.

Um arquivo por declaração: não declarar ou implementar mais de uma classe, interface, type, etc. no mesmo arquivo (cada tipo/objeto deve possuir seu próprio ficheiro).
